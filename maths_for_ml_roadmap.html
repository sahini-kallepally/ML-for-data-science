Linear Algebra is the language of data in ML. Every dataset, image, or model is stored as vectors and matrices. This course teaches you to understand and manipulate them.
1. Vectors
Core▶
What is a vector?
A vector is a list of numbers. In ML, it represents one data point — one person, one image, one word.
v = [2, 3]      # 2D vector — point in 2D space
v = [1, 0, 5]   # 3D vector — point in 3D space
np.array([2, 3])  # shape: (2,) — 1D, flat
Vector operations
Addition, scaling, and dot product are the three core operations. 
a = np.array([1, 2])
b = np.array([3, 4])

a + b        # → [4, 6]   add element by element
2 * a        # → [2, 4]   scale every element
np.dot(a,b)  # → 11  (1×3 + 2×4) — dot product
Dot product
Multiply matching elements, then add. Result is a single number. Measures how similar two vectors are. 
# dot product by hand
a = [1, 2, 3]
b = [4, 5, 6]
# (1×4) + (2×5) + (3×6) = 4+10+18 = 32
Vector norm (length)
The length of a vector. Square root of sum of squares. 
v = [3, 4]
np.linalg.norm(v)  # → 5.0  (√(9+16))
Used in ML for: representing data points, word embeddings, measuring similarity between items (cosine similarity)
2. Matrices
Core▶
What is a matrix?
A 2D array of numbers. Rows and columns. Shape = (rows, cols). Think of it as a table of data. 
A = np.array([[1, 2, 3],
              [4, 5, 6]])
A.shape  # → (2, 3) — 2 rows, 3 columns
# Rule: shape = (rows, columns) — "RC Cola"
Matrix operations
A + B        # element-wise addition (same shape)
A * 2        # multiply every element by 2
A @ B        # matrix multiplication ← most important!
A.T          # transpose — flip rows and columns
Matrix multiplication
Row of A × column of B → each element of result. 
A = [[2, 3],   B = [1,    A@B:
     [2, 1]]        0]    row0: 2×1+3×0=2
                          row1: 2×1+1×0=2
                          result: [2, 2]
Special matrices
np.eye(3)       # identity matrix — I (like number 1 for matrices)
np.zeros((3,3)) # all zeros
np.ones((2,3))  # all ones
Used in ML for: storing entire datasets (m examples × n features), representing neural network weights, transformations
3. Systems of Linear Equations
Key▶
What is it?
Multiple equations, multiple unknowns. Pack them into a matrix and solve together. 
# 2 equations, 2 unknowns: x and y
# -x + 3y = 7
#  3x + 2y = 1
A = np.array([[-1, 3],
              [ 3, 2]])
b = np.array([7, 1])
Augmented matrix
Glue A and b together into one matrix to work with: 
A_sys = np.hstack((A, b.reshape(2,1)))
# → [[-1  3  7]
#    [ 3  2  1]]
Gaussian elimination
Use row operations to create staircase (zeros below diagonal). Then read answers bottom to top. 
# 3 tools:
SwapRows(A, i, j)       # swap two rows
MultiplyRow(A, i, k)    # multiply row by scalar
AddRows(A, i, j, k)     # k×row_i + row_j → row_j

# Goal: get this staircase shape
# [■  ■  ■ | ■]
# [0  ■  ■ | ■]
# [0  0  ■ | ■]  ← read x3 directly!
Number of solutions
# det ≠ 0  → exactly 1 solution  ✅
# det = 0, inconsistent → 0 solutions ❌
# det = 0, consistent   → ∞ solutions ♾️
np.linalg.det(A)  # check determinant
Used in ML for: solving regression problems, finding optimal parameters
4. Determinant, Rank & Singularity
Key▶
Determinant
A single number that tells if a matrix has a unique solution. 
A = np.array([[2, 3],
              [1, 4]])
# by hand: (2×4) - (3×1) = 8-3 = 5
np.linalg.det(A)  # → 5.0
# det ≠ 0 → invertible → 1 solution ✅
# det = 0 → singular  → no/∞ solutions ❌
Rank
Number of rows that carry unique (independent) information. Found by row reduction — count non-zero rows. 
A = np.array([[1, 2],
              [2, 4]])  # row2 = 2×row1 — no new info!
np.linalg.matrix_rank(A)  # → 1

B = np.array([[1, 2],
              [3, 4]])  # rows independent
np.linalg.matrix_rank(B)  # → 2 (full rank)
Singularity
# Singular matrix:     det = 0, rank < n, NOT invertible
# Non-singular matrix: det ≠ 0, rank = n, IS invertible
# rank = n (full rank) → same as det ≠ 0 → 1 unique solution
Used in ML for: checking if a model can find a unique solution, detecting redundant features in data
5. Matrix Inverse & Linear Transformations
Key▶
Matrix inverse
Like dividing by a matrix. A⁻¹ × A = I (identity). Only exists when det ≠ 0. 
A = np.array([[2, 3],
              [1, 4]])
A_inv = np.linalg.inv(A)
A_inv @ A  # → [[1,0],[0,1]] = identity ✅

# Solve Ax = b using inverse:
x = np.linalg.inv(A) @ b
Linear transformations
Matrix multiplication is a transformation — it rotates, scales, shears vectors. 
A = np.array([[2, 0],
              [0, 3]])
v = np.array([1, 1])
A @ v  # → [2, 3] — stretched x by 2, y by 3

# common transforms:
# scaling, rotation, reflection, shear
Used in ML for: solving Ax=b directly, understanding how neural networks transform data through layers
6. Eigenvalues & Eigenvectors
Applied▶
What are they?
Special vector v and number λ where A×v = λ×v. The matrix only stretches v — never rotates it. 
A = [[2, 3],
     [2, 1]]
# A × v = λ × v
# eigenvector v just gets scaled by λ, no direction change
Finding eigenvalues (math)
# Step 1: det(A - λI) = 0
# Step 2: expand → get quadratic in λ
# Step 3: solve quadratic → get λ₁, λ₂

A - λI = [[2-λ,  3],
          [ 2,  1-λ]]
det = (2-λ)(1-λ) - 6 = λ²-3λ-4 = 0
→ λ₁=4, λ₂=-1
Finding eigenvectors (math)
# For each λ, solve (A-λI)v = 0
# λ=4: [[-2,3],[2,-3]] × v = 0
#       -2x+3y=0 → x=3,y=2 → v₁=[3,2]
# λ=-1: [[3,3],[2,2]] × v = 0
#        x+y=0 → x=-1,y=1 → v₂=[-1,1]

vals, vecs = np.linalg.eig(A)  # numpy way
Used in ML for: PCA (dimensionality reduction), understanding how data spreads, compressing data while keeping important patterns
7. NumPy Essentials
Applied▶
Array creation & shapes
np.array([1,2,3])        # 1D: shape (3,)
np.array([[1,2],[3,4]])  # 2D: shape (2,2)
np.zeros((3,3))          # all zeros (initialized)

np.arange(0,1,0.25)      # [0, 0.25, 0.5, 0.75] — you set step

Indexing & slicing
a[0]       # first row → 1D (dimension drops!)

a[:, 1]    # column 1 → 1D
a[:, 1:2]  # column 1 → 2D (use this for ML!)

Stacking & splitting
np.vstack((a, b))    # stack rows (↓) — same cols needed
np.hstack((a, b))    # stack columns (→) — same rows needed
np.hsplit(a, 3)      # split into 3 equal column chunks
np.hsplit(a, [2,4])  # split before col 2 and col 4
Broadcasting
NumPy auto-expands smaller arrays to match larger ones. Shapes compatible if equal or one is 1. 
a = np.array([[1,2,3],[4,5,6]])  # (2,3)
b = np.array([10,20,30])         # (3,) → acts like (1,3)
a + b  # → [[11,22,33],[14,25,36]] — b broadcast across rows
Used in ML for: all data manipulation, feeding data into models, reshaping between operations
Calculus teaches how to optimize — find the minimum of a cost function. This is the mathematical engine behind training all neural networks.
1. Derivatives
Core▶
What is a derivative?
The slope of a function at any point. How fast is output changing when input changes a tiny bit? 
# f(x) = x²
# f'(x) = 2x  ← derivative

# at x=3: slope = 2×3 = 6  (going up steeply)
# at x=0: slope = 0         (at the bottom — minimum!)
Basic rules
# Power rule:   d/dx[xⁿ] = n×xⁿ⁻¹
# x³ → 3x²,   x² → 2x,   x → 1

# Constant:    d/dx[c] = 0
# Sum:         d/dx[f+g] = f' + g'
# Chain rule:  d/dx[f(g(x))] = f'(g(x)) × g'(x)
Finding minimum
At minimum, slope = 0. Set derivative = 0 and solve. 
# f(x) = x² - 4x + 3
# f'(x) = 2x - 4 = 0
# x = 2  ← minimum is at x=2
Used in ML for: finding where the cost function is minimum — that's where the best W and b are!
2. Gradient Descent
Core▶
What is it?
An algorithm that finds the minimum of a function by taking small steps in the downhill direction. This is HOW neural networks learn. 
# Imagine a valley ⛰️:
# Current position = current W and b
# Goal = roll down to lowest point (min cost)
# Gradient = direction of steepest uphill
# Step = move in OPPOSITE direction (downhill)
Update rule
W = W - learning_rate × dL/dW
b = b - learning_rate × dL/db

# learning_rate: how big each step is
# dL/dW: gradient — which direction is uphill?
# minus sign: we go DOWNHILL (opposite of gradient)
Learning rate
# too large → overshoots minimum, bounces around ❌
#  0.1  →  [big jumps, unstable]
# too small → takes forever to converge ❌
#  0.0001 → [tiny steps, too slow]
# just right → smooth convergence ✅
#  0.01  →  [typical starting value]
Used in ML for: training every neural network — adjusting W and b in the right direction every iteration
3. Cost Functions
Core▶
What is a cost function?
A number that measures how wrong your model's predictions are. Training = minimizing this number. 
# Mean Squared Error (for regression):
# L = (1/2m) × Σ(ŷ - y)²
#
def compute_cost(Y_hat, Y):
    m = Y.shape[1]
    cost = np.sum((Y_hat - Y)**2) / (2*m)
    return cost
# Perfect predictions → cost = 0 ✅
# Terrible predictions → cost = huge ❌
Why squared error?
# Without squaring: +2 and -2 errors cancel out → misleading!
# With squaring:    4 and 4 → always positive → honest total error
# ÷2m → average (not total), makes derivative cleaner
Used in ML for: measuring model quality, guiding gradient descent toward better parameters
4. Partial Derivatives & Gradients
Key▶
Partial derivatives
When a function has multiple inputs (like W and b), you take the derivative with respect to ONE at a time. 
# L(W, b) = cost function with 2 parameters
# ∂L/∂W = how much does cost change if W changes slightly?
#          (treat b as constant)
# ∂L/∂b = how much does cost change if b changes slightly?
#          (treat W as constant)
Gradient vector
Pack all partial derivatives into one vector. Points in direction of steepest increase. 
# ∇L = [∂L/∂W,  ∂L/∂b]
#        ↑         ↑
#     gradient  gradient
#     w.r.t W   w.r.t b
# gradient descent: move OPPOSITE to ∇L
Used in ML for: backward propagation — computing how much each weight contributed to the error
5. Forward & Backward Propagation
Applied▶
Forward propagation
Feed input forward through the network to get a prediction. 
def forward_propagation(X, parameters):
    W = parameters["W"]
    b = parameters["b"]
    Z = np.matmul(W, X) + b  # Z = WX + b
    Y_hat = Z                 # prediction
    return Y_hat
# Shape: W(1,1) × X(1,30) + b(1,1) = Y_hat(1,30)
Backward propagation
Compute gradients and update W and b. Uses chain rule from calculus. 
# dL/dW = (1/m) × (Y_hat - Y) × X.T
# dL/db = (1/m) × sum(Y_hat - Y)
# Then update:
W = W - lr × dL/dW
b = b - lr × dL/db
Full training loop
def nn_model(X, Y, num_iterations=1000):
    parameters = initialize_parameters(n_x, n_y)
    for i in range(num_iterations):
        Y_hat = forward_propagation(X, parameters)  # predict
        cost  = compute_cost(Y_hat, Y)              # measure error
        parameters = update_params(parameters, ...)  # adjust W,b
    return parameters  # best W and b found!
Used in ML for: this IS how neural networks learn — the entire training process
Probability and Statistics let you understand uncertainty in data and make confident predictions. Essential for understanding model outputs, distributions, and evaluation.
1. Probability Basics
Core▶
What is probability?
A number between 0 and 1 measuring how likely an event is. 0 = impossible, 1 = certain. 
# P(heads) = 0.5
# P(rain) = 0.3  (30% chance)
# P(A and B) = P(A) × P(B)  if independent
# P(A or B)  = P(A) + P(B) - P(A and B)
Conditional probability
Probability of A given B already happened. 
# P(A|B) = P(A and B) / P(B)
# "Given it's cloudy, what's P(rain)?"
# P(rain|cloudy) = P(rain and cloudy) / P(cloudy)
Bayes' theorem
Update your belief when you get new evidence. 
# P(A|B) = P(B|A) × P(A) / P(B)
# Example: spam filter
# P(spam|"free money") = P("free money"|spam) × P(spam)
#                        ──────────────────────────────
#                              P("free money")
Used in ML for: Naive Bayes classifiers, spam detection, understanding model confidence
2. Probability Distributions
Core▶
Normal (Gaussian) distribution
The famous bell curve. Most data clusters around the mean. Described by μ (mean) and σ (std dev). 
import numpy as np
data = np.random.normal(mean=0, std=1, size=1000)
# 68% of data within 1σ of mean
# 95% of data within 2σ of mean
# 99.7% within 3σ — the "68-95-99.7 rule"
Binomial distribution
Count of successes in n trials, each with probability p. 
# Flip coin 10 times. Count heads.
# n=10 trials, p=0.5 probability
# P(exactly k heads) = binomial formula
import scipy.stats as stats
stats.binom.pmf(k=6, n=10, p=0.5)  # P(6 heads)
Used in ML for: weight initialization (normal dist), classification outputs, modeling real-world data
3. Descriptive Statistics
Core▶
Central tendency
data = [2, 4, 4, 6, 8]
np.mean(data)    # → 4.8   (average)
np.median(data)  # → 4.0   (middle value)
# mode = most frequent value
Spread / variability
np.std(data)     # → std deviation (avg distance from mean)
np.var(data)     # → variance = std²
np.min(data)     # → 2
np.max(data)     # → 8
np.percentile(data, 25)  # → Q1 (25th percentile)
Normalization
Scale data to the same range so features don't dominate each other. 
# Z-score normalization (mean=0, std=1):
X_norm = (X - np.mean(X)) / np.std(X)
# Why? House size (1000s) vs quality (1-10)
# Without normalization: size dominates learning!
# After normalization: both treated equally ✅
Used in ML for: preprocessing data before training, understanding feature distributions
4. Statistical Inference
Key▶
Population vs sample
# Population: ALL possible data points (often impossible to get)
# Sample: subset of population you actually have
# Goal: use sample statistics to estimate population parameters

# Sample mean  → estimates → population mean
# Sample std   → estimates → population std
Confidence intervals
A range that likely contains the true population value. 
# 95% confidence interval:
# mean ± 1.96 × (std / √n)
# "We're 95% confident the true mean is in this range"
# n = sample size — larger n → narrower interval ✅
Hypothesis testing
Is an observed effect real or just random chance? 
# H₀ (null hypothesis): "no effect" — default assumption
# H₁ (alt hypothesis):  "there IS an effect"
# p-value < 0.05 → reject H₀ → effect is real ✅
# p-value > 0.05 → fail to reject H₀ → could be chance
Used in ML for: A/B testing model variants, validating that model improvements are statistically significant
5. Regression & Model Evaluation
Applied▶
Linear regression
Fit a line ŷ = Wx + b to data. Find W and b that minimize error. 
# Simple:   ŷ = wx + b       (1 input)
# Multiple: ŷ = w₁x₁+w₂x₂+b (2+ inputs)
# In matrix form: Ŷ = WX + b
# W shape: (n_y, n_x), X shape: (n_x, m)
# Result Ŷ shape: (n_y, m) — one prediction per example
Evaluation metrics
# MSE = mean((ŷ - y)²)        → lower is better
# RMSE = √MSE                 → same units as y
# R² = 1 - SS_res/SS_tot      → 1.0 = perfect fit
#                                0.0 = as bad as just using mean
Overfitting vs underfitting
# Underfitting: model too simple — high error on training data
# Overfitting:  model memorizes training — fails on new data
# Goal: generalize — perform well on UNSEEN data
# Solution: train/test split, regularization, cross-validation
Used in ML for: this IS machine learning — training, evaluating, and deploying predictive models
